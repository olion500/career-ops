# 가우스랩스


# 직무 설명

## 업무

DevOps/Infrastructure 엔지니어로서 복잡한 인프라 관리:

1. **CI/CD 파이프라인 설계 및 관리**
   - 여러 애플리케이션/서비스를 위한 파이프라인
   - 자동화된 배포

2. **Infrastructure-as-Code (IaC)**
   - Terraform을 사용한 IaC 구축
   - 코드 기반 인프라 관리

3. **Kubernetes (EKS) 클러스터 관리**
   - AWS EKS 클러스터 운영
   - 컨테이너 오케스트레이션
   - 배포 전략 수립

4. **복잡한 문제 해결**
   - 시스템, 인프라, 애플리케이션 계층 문제 해결
   - 애플리케이션 코드까지 분석
   - 로그 및 메트릭 분석

## 작업요건

- DevOps/Infrastructure 전문 지식
- 대규모 시스템 운영 경험
- 문제 해결 능력
- 영어 소통 능력 (필수)
- Terraform, Kubernetes, Docker, AWS 경험

## 평가 & 분석

### 강점
- **높은 평가**: 잡플 4.6, 블라 4.2 (높음)
- **글로벌 경험**: 국제 팀과의 협업
- **현대 기술스택**: Terraform, Kubernetes 등 최신 기술
- **반도체/AI**: 미래 기술 분야
- **영어 사용**: 국제 경험 및 네트워크

### 약점
- **경영진 소통 문제**: 경영자들이 소통을 잘 안 한다는 평가
- **정보 부족**: 회사 상세 정보 미확인
- **글로벌 회사**: 문화 차이 있을 수 있음

### 최종 평가
**참고**:
- 기술적으로는 매력적
- 평가점수도 높음
- 다만 경영진과의 소통 문제가 우려
- 더 많은 정보 수집 필요
